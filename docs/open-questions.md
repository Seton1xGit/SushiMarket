## A. WhatsApp / отправка (самый длинный блок)

1. **Какой стек отправки выбираем?**
   - **A1.** WhatsApp Cloud API (Meta, официальный): ~$5–10/мес, риск бана = 0, ждать 1–7 дней верификации Meta + согласования шаблонов.
   - **A2.** Свой `wa-bridge` (whatsapp-web.js): бесплатно, риск бана номера, ставится за час, поддерживать самостоятельно.
   - **A3.** GreenAPI / Whapi.cloud: ~$10–25/мес, готовая обёртка над A2, без своей инфры.
2. **Если A2 (свой wa-bridge):** где хостить?
   - тот же сервер, где `n8n.thefreedom.pro` — нужен доступ (Docker / SSH);
   - отдельный VPS (Oracle Free всегда бесплатно / Hetzner ~€4/мес);
   - локально (требует постоянно включённого ПК — не вариант).
3. **Если A2:** есть отдельный SIM/номер для бизнеса (не личный)?
4. **Принимаем ли риск бана номера?** Для транзакционных уведомлений (1 на заказ) риск низкий, но ненулевой.
5. **Если A1 (Cloud API):** есть готовый Meta Business аккаунт + WABA, или поднимаем с нуля?

## D. Поведение системы при первом запуске

16. **«Каскад приветствий» при первом старте.** В момент включения WF-A видит N активных заказов в админке, для каждого порождает `new_order` → каждому летит welcome. Но эти заказы уже могут быть оформлены час назад. Нужно ли вводить grace-период (отправлять welcome только заказам моложе X минут)?
17. **Стартовая сидинговая прогонка** перед активацией: пройтись polling один раз, заполнить order_events существующими активными заказами без отправки сообщений. Делаем?
